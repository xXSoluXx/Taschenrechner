# Analyse

## Benutzeranforderungen
- Benutzer kann zwei Zahlen eingeben  
- Benutzer kann eine Rechenoperation auswählen (+, -, *, /)  
- Benutzer sieht das Ergebnis der Berechnung  
- Benutzer bekommt eine Fehlermeldung bei ungültigen Eingaben  

## Funktionale Anforderungen
- System muss Eingaben vom Benutzer entgegennehmen  
- System muss die gewählte Rechenoperation korrekt ausführen  
- System muss das Ergebnis berechnen und anzeigen  
- System muss Fehler erkennen (z. B. Division durch 0 oder falsche Eingaben)  

## Nicht-funktionale Anforderungen
- Performance: Berechnungen sollen sofort erfolgen (keine spürbare Verzögerung)  
- Benutzerfreundlichkeit: Einfache und verständliche Eingabe über die Konsole  
- Sicherheit: Programm darf nicht abstürzen bei falschen Eingaben  

## Edge Cases
- Division durch 0  
- Benutzer gibt Text statt Zahlen ein  
- Benutzer gibt keine Eingabe ein  
- Sehr große oder sehr kleine Zahlen
